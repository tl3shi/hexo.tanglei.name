配置状态报告

目的:配置状态报告 (Configuration Status Report)有效记录和报告管理配置所需要的信息，及时、准确地给出软件配置项的当前状态，供相关人员了解，以加强配置管理工作。需要定期提交配置状态报告.

主要内容


  1. 各份变更请示概要：变更请求号、日期、申请人、状态、估计工作量、实际工作量、发行版本、变更结束日期

  2. 基线库状态：库标识、至某日预计库内配置项数、实际配置项数

  3. 发行信息：发行版本、计划发行时间、实际发行日期、说明

  4. 备份信息：备份日期、介质、备份存放位置

  5. 配置管理工具状态

  6. 配置管理培训状态

配置审计报告

目的：验证配置项信息与配置标识（需求、标准、流程…）的一致性，4″W”（Who、When、Why、What）。

审计内容


  1. 配置项状态统计

  2. 基线库基线统计

  3. 变更统计

  4. 审计中发现的主要问题
